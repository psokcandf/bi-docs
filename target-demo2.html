Expected behavior

  1. Before consent, Adobe Target should not load.
  2. After the configured Target consent category is granted, the wrapper waits for the MetaRouter anonymous ID.
  3. The wrapper passes that ID to Target as `mbox3rdPartyId`, then injects `at.js`.
  4. When Target returns a decision, the bridge reads response tokens.
  5. The bridge sends normalized properties to `metaTagger.track()` for MetaRouter mapping.

Target identity stitching

The identity stitching layer is separate from the Target exposure payload. MetaRouter keeps sending `experience_name`, `experience_type`, and `event222` exactly as before. The additional identity value is used so Treasure Data CDP can update the matching Adobe Target profile later.

Console and Network validation

After accepting the Target OneTrust category, use these checks to confirm that Target loaded, the MetaRouter ID was passed into Target, and the activity response was captured.

Console checks

// Adobe Target loaded adobe?.target?.VERSION // Target params sent before at.js requests window.targetPageParamsAll() // MetaRouter source ID and Target identity state analytics.user().anonymousId() window.AdobeTargetIdentity.getState() // Target exposure events captured by this demo window.TargetAnalyticsBridge.getQueue()

Network checks

  1. Open DevTools > Network.
  2. Filter by `delivery` or by the Adobe Target host, for example `tt.omtrdc.net`.
  3. Open the Target `delivery` request after consent is accepted.
  4. Check the request payload for `property.token` and `id.thirdPartyId`.
  5. Check the response for `responseTokens`, especially `activity.name` and `experience.name`.
// Expected request values // Browser input name targetPageParamsAll().mbox3rdPartyId: "SAME_VALUE_AS_analytics.user().anonymousId()" // Adobe Target Delivery API request shape property.token: "PROPERTY_TOKEN_FROM_ADOBE_TARGET" id.thirdPartyId: "SAME_VALUE_AS_analytics.user().anonymousId()" // Expected response tokens when an activity qualifies activity.name: "Target Activity Name" experience.name: "Target Experience Name"
